Atue como um Arquiteto de Software e Desenvolvedor Full-Stack Sênior, especialista em aplicações web "vanilla" (HTML/CSS/JS puros) e arquiteturas serverless.

Objetivo Principal:
Criar um website empresarial completo para uma consultoria de tecnologia (ex: "Pinheiro Tecnologia"). O site deve ser dinâmico, responsivo e gerenciável por um painel de administração.

CRÍTICO: Simulação de Backend (Stubbing):
Este projeto é apenas frontend. Você NÃO DEVE construir o backend (AWS, DynamoDB, Cognito). Em vez disso, você deve SIMULAR todo o comportamento do backend dentro do script.js:

Dados Fictícios: Crie uma constante global MOCK_PROJECTS (um array de objetos) que contenha 7-8 projetos fictícios. Cada projeto deve ter: id, title, category (ex: 'data-analysis', 'apps', 'automation'), thumbnailSrc, iframeSrc, embedTitle, tabsToShow (string), e um objeto data com o HTML para as 6 abas do modal.

Simulação de API: Todas as funções que buscam ou enviam dados (ex: fetchProjects, handleProjectSubmit) devem ser async, mas devem usar os MOCK_PROJECTS no bloco catch (simulando uma falha de API) ou após um setTimeout para popular a página.

Simulação de Login: O login.html deve ter sucesso com qualquer e-mail e senha. Ele deve apenas guardar um token falso (ex: 'fake-token') no localStorage e redirecionar para admin.html.

Simulação de Auth: A admin.html deve ser protegida verificando a existência desse token falso. O botão "Sair" deve limpá-lo.

Pilha de Tecnologia (Obrigatória):

HTML5 Semântico

CSS3 Puro: Obrigatoriamente usar Variáveis CSS (:root).

JavaScript (ES6+) Puro: Um único arquivo script.js unificado.

NÃO USAR: Frameworks (React, Vue), bibliotecas (jQuery) ou pré-processadores.

Estrutura de Arquivos (Gerar todos):

index.html (Página Principal / Hero / Serviços / Cases em Destaque)

projetos.html (Nova página - Portfólio Completo por Categoria)

contato.html (Página de Contato com formulário e links)

README.md (Documentação do projeto)

backend-architecture.md (Guia explicando a arquitetura AWS real que foi simulada)

Requisitos Detalhados de style.css:

Tema: Limpo, profissional, "light-mode".

Cores (:root):

--primary-color-accessible: #1A6A6C

--background-color: #f4f7f6

--surface-color: #ffffff

--text-color: #333333

Responsividade: Design "Mobile-first".

Navegação: Um .main-nav não-fixo (position: relative) que rola com a página. Deve aparecer em TODAS as 6 páginas.

Scroll Snap: NÃO implementar Scroll Snapping. A rolagem da página deve ser livre.

Cartões: Estilo de cartão consistente para serviços (.service-card) e projetos (.project-card).

Layout (Flexbox): Usar display: flex e justify-content: center no .project-grid (na página projetos.html) para que os cartões de projeto fiquem centralizados na seção.

Tamanhos: O título da seção (.section-title) deve ter 2.0rem e os cartões de projeto (.project-card) devem ter max-width: 340px.

Requisitos Detalhados das Páginas (HTML + script.js):

1. index.html (Página Principal):

Uma seção "Hero" (.hero-section) com título e CTA.

Uma seção "Nossos Serviços" (.services-grid).

Uma seção "Nossos Cases de Sucesso" (.project-category):

Deve ter um grid vazio (id="project-grid-dynamic") dentro de um div wrapper (.project-grid-index-wrapper).

O CSS para .project-grid-index-wrapper deve forçar uma linha única horizontal (flex-wrap: nowrap) e ocultar o transbordamento (overflow-x: hidden).

O script.js (na initIndexPage) deve chamar fetchProjectsForIndex para preencher este grid com no máximo 6 projetos dos MOCK_PROJECTS.

Deve incluir um botão "Ver Todos os Projetos" (<a href="projetos.html">) no final.

Uma seção "Empresas que Confiam em Nós" (.clients-section):

Deve ser um carrossel de logos infinito, responsivo.

O script.js (na initClientCarousel) deve duplicar os logos para garantir o loop da animação CSS.

2. projetos.html (Página de Portfólio):

Página nova que lista TODOS os projetos.

Deve ter um título principal (ex: "Nosso Portfólio de Projetos").

Deve ter múltiplas seções, uma para cada categoria (ex: "Projetos de Análise de Dados", "Projetos de Aplicativos", "Projetos de Automação").

Cada seção de categoria deve ter seu próprio grid (id="grid-data-analysis", id="grid-apps", etc.) e seu próprio loader (id="loader-data-analysis").

O script.js (na initProjetosPage) deve chamar fetchProjectsForCategorization, que por sua vez chama distributeProjects. Esta função deve filtrar os MOCK_PROJECTS por category e preencher os grids corretos.

3. contato.html (Página de Contato):

No topo, deve ter um formulário de contato (id="contact-form") dentro de um wrapper (.contact-form-wrapper).

Abaixo do formulário, deve ter uma seção "Outros Canais" (.contact-channels-section) com cartões (.contact-card) para LinkedIn, Email e WhatsApp.

4. login.html (Página de Login):

Deve ter um formulário de login (id="login-form") com campos para e-mail e senha.

O script.js (na handleLoginSubmit) deve simular o login, guardar um token falso no localStorage e redirecionar para admin.html.

5. admin.html (Painel de Admin):

O script.js (na checkAdminAuth) deve verificar o token falso no localStorage; se não existir, deve redirecionar para login.html.

Deve ter um botão "Sair" (id="logout-button").

Deve ter um formulário (id="project-form") para Adicionar/Editar projetos.

O formulário DEVE incluir:

project-id (hidden field).

project-title (text).

project-category (um <select> com as opções: 'data-analysis', 'apps', 'automation', 'other').

project-thumbnail (url).

project-iframe-src (url).

6 campos <textarea> para o conteúdo das 6 abas (ex: id="tab-descricao", id="tab-objetivos").

Deve ter uma lista (id="project-list-container") para exibir, editar e excluir os projetos carregados dos MOCK_PROJECTS.

6. script.js (Lógica Global):

Roteamento de Página: Usar um DOMContentLoaded que verifica window.location.pathname para chamar a função init correta (ex: initIndexPage, initProjetosPage, initAdminPage, etc.).

Modal Global:

A lógica do modal (listeners, openModal, closeModal, handleTabClick) deve ser global e reutilizável por index.html e projetos.html.

openModal(project): Deve preencher dinamicamente o título, o iframe.src e o HTML de todas as abas, com base no objeto project clicado.

O modal deve ter height: 95vh e o iframe-container deve ter height: 60vh para garantir que as abas inferiores sejam sempre visíveis.

Deve incluir acessibilidade (Focus Trap e fechar com 'Escape').

Funções de Renderização:

populateProjectGrid(gridElementId, projects): Função reutilizável que constrói os .project-card e os anexa ao grid.

distributeProjects(projects): Função (para projetos.html) que filtra os projetos por categoria e chama populateProjectGrid para cada categoria.
